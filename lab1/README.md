# Лабораторная работа №1: Модель пылесоса

**Документация системы имитации работы пылесоса**

## 1. Общее описание

Данная программная система представляет собой консольное приложение (CLI) для моделирования работы бытового пылесоса. Система позволяет управлять компонентами устройства (мотор, контейнер, фильтр, насадки), проводить уборку помещений, выполнять техническое обслуживание и сохранять состояние между запусками.

Проект реализован на языке Python с использованием объектно-ориентированного подхода, демонстрируя принципы инкапсуляции, композиции и обработки исключений.

## 3. Описание модулей

### 3.1. Пакет core

#### VacuumCleaner.py

Содержит класс `VacuumCleaner`, который является агрегатором всех компонентов системы.

- Управляет жизненным циклом компонентов (`Motor`, `DustContainer`, `Filter`).
- Предоставляет методы для взаимодействия с внешним миром: `clean_room` (уборка), `maintenance` (обслуживание), `change_attachment` (смена насадки).
- Реализует сохранение и загрузку состояния через методы `save_state` и `load_state`.

#### Компоненты устройства

- **Motor**: Управляет состоянием питания (вкл/выкл) и уровнями мощности (1-3).
- **DustContainer**: Хранит собранную пыль. Имеет фиксированную вместимость. Выбрасывает исключение `ContainerFullException`, если переполнен.
- **Filter**: Отслеживает уровень износа. При достижении 100% требует замены, иначе блокирует работу пылесоса (`FilterDegradedException`).
- **Attachment**: Сменные насадки (например, "Стандартная щетка", "Узкая насадка").

#### Система управления (Buttons)

Реализована через паттерн, напоминающий "Команду".

- **ControlButton**: Абстрактный базовый класс.
- **PowerButton**: Инкапсулирует логику включения/выключения мотора.
- **ModeButton**: Инкапсулирует логику переключения режимов мощности.

#### Room.py

Модель помещения. Содержит название и текущее количество пыли. Метод `accumulate_dust` имитирует естественное загрязнение комнаты со временем.

#### exceptions.py

Определяет иерархию ошибок для обработки штатных и нештатных ситуаций:

- **VacuumException** (базовый)
- **ContainerFullException** (контейнер полон)
- **FilterDegradedException** (фильтр изношен)
- **MotorStateError** (попытка включить уже включенный мотор и т.д.)

### 3.2. CLI (main.py)

Точка входа в приложение.

- Загружает состояние пылесоса и список комнат при старте.
- Предоставляет текстовое меню из 11 пунктов для полного управления устройством.
- Обрабатывает исключения, возникающие в процессе работы (например, попытка убрать комнату с полным контейнером), и выводит понятные сообщения пользователю.
- Автоматически сохраняет состояние при выходе.

### 3.3. Тестирование (tests/)

Набор модульных тестов на базе библиотеки `unittest`.

- Проверяет корректность работы отдельных компонентов.
- Тестирует граничные условия (переполнение, очистка, сериализация).

## 4. Ключевые алгоритмы и реализация

### Инкапсуляция и Свойства

В классах активно используются приватные поля (с префиксом `_`) и декоратор `@property` для доступа к ним. Это защищает внутреннее состояние (например, текущую заполненность контейнера) от некорректного прямого изменения.

### Алгоритм уборки (clean_room)

1. **Проверки**: Пылесос должен быть включен, контейнер не полон, фильтр не изношен.
2. **Расчет мощности**: Эффективность уборки зависит от уровня мощности мотора (`power_level * 35`).
3. **Сбор пыли**: Количество собираемой пыли рассчитывается как минимум из трех величин:
   - Текущее количество пыли в комнате.
   - Свободное место в контейнере.
   - Мощность всасывания.
4. **Побочные эффекты**:
   - Увеличивается заполненность контейнера.
   - Уменьшается количество пыли в комнате.
   - Увеличивается износ фильтра (пропорционально собранной пыли).

### Сериализация

Каждый ключевой класс реализует методы `to_dict` и `from_dict`. Это позволяет рекурсивно преобразовать сложный объект `VacuumCleaner` в JSON-структуру и восстановить его обратно, сохраняя состояние всех вложенных объектов (мотора, фильтра, списка насадок).

## 5. Инструкция по запуску

### Запуск приложения

Находясь в корневой директории проекта (`lab1`), выполните команду:

```bash
python main.py
```

### Запуск тестов

Для запуска всех unit-тестов выполните:

```bash
python -m unittest discover tests
```
