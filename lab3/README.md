# Лабораторная работа №3: Игра "Шахматы"
**Документация системы классической настольной игры**

## 1. Общее описание
Данная программная система представляет собой полноценный симулятор игры в шахматы. Проект реализован с использованием графической библиотеки Pygame и предлагает полный игровой цикл: главное меню, локальную игру на одном устройстве, сетевую онлайн-игру для двух пользователей, систему динамических подсказок ходов, а также таблицу рекордов победителей.

Проект построен на строгом соблюдении архитектурного паттерна **MVC (Model-View-Controller)** и принципа единственной ответственности (Single Responsibility). Логика игры (шахматные правила), отрисовка графики, обработка пользовательского ввода и сетевое взаимодействие полностью изолированы друг от друга, что делает архитектуру гибкой и легко расширяемой.

## 2. Структура проекта
Файловая структура проекта организована следующим образом:

```text
Lab3/
├── server.py               # Точка входа для сервера (для онлайн-игры)
├── data/                   # Динамические данные (сохраняются программой)
│   ├── config.json         # Файл с цветовыми настройками доски
│   └── leaderboard.json    # Файл сохранения рекордов игроков
├── assets/                 # Статичные медиа-ресурсы
│   ├── images/             # Спрайты шахматных фигур
│   ├── sounds/             # Аудиоэффекты и фоновая музыка
│   └── rules.txt           # Текстовый файл со справкой (правилами)
└── source/                 # Исходный код клиента
    ├── main.py             # Точка входа в игру, контроллер состояний (State Machine)
    ├── model/          # Математическая логика и правила шахмат (Model)
    ├── view/            # Отрисовка графики и анимаций (View)
    ├── GameRenderer.py       # Обработка кликов и связи с сетью (Controller)
    ├── ui/           # Менеджер пользовательского интерфейса (кнопки, меню)
    ├── network.py          # Сетевой клиент для сокетного соединения
    └── managers/           # Вспомогательные сервисы
        ├── Config_manager.py # Загрузчик настроек
        ├── Record_manager.py # Управление таблицей рекордов
        └── Sound_manager.py  # Управление аудиоплеером Pygame
```

## 3. Описание модулей
### 3.1. Ядро игры (MVC)
- **model**: Содержит математическую модель доски (`Board`) и иерархию фигур (`Piece`, `Pawn`, `Knight` и др.). Отвечает за генерацию псевдолегальных ходов и фильтрацию легальных ходов (с учетом шаха, взятия на проходе, рокировки и превращения пешек).
- **view (`GameRenderer`)**: Отвечает за всю графику. Отрисовывает доску, координаты, фигуры, подсказки для ходов. Содержит логику плавных покадровых анимаций скольжения фигур и растворения (fade-out) при взятии.
- **GameController**: Связующее звено. Получает координаты клика мыши, конвертирует их в логические координаты доски, запрашивает у модели возможность хода и дает команду `view` на запуск анимации, а `sound_manager` — на воспроизведение звука.

### 3.2. Пользовательский интерфейс и Сеть
- **ui (`UIManager`)**: Создает и позиционирует элементы управления (`Button`, `InputBox`). Содержит логику умного переноса текста (word-wrap) и вертикального скроллинга для экрана "Справка".
- **network.py (`Network`)**: Инкапсулирует логику работы с TCP-сокетами. Выполняет подключение к серверу и запускает фоновый поток (`threading.Thread`) для прослушивания ходов соперника, не блокируя основной цикл игры.
- **server.py**: Легковесный TCP-сервер, который ожидает подключения ровно двух клиентов, раздает им цвета (белые/черные) и ретранслирует строковые сообщения с координатами ходов между ними.

### 3.3. Вспомогательные менеджеры (`source/managers/`)
- **SoundManager**: Загружает `.wav` и `.mp3` файлы. Имеет защиту от сбоев (если файла нет, игра продолжает работу без звука). Управляет переключением фоновой музыки между меню и самой игрой.
- **RecordManager**: Записывает имена победителей и количество их побед в JSON, автоматически сортируя топ-10 игроков.

## 4. Ключевые алгоритмы и реализация
### Валидация легальных ходов (Virtual Board Check)
Для предотвращения ходов, оставляющих или подставляющих собственного короля под шах, алгоритм в `model.py` перед выдачей списка доступных клеток выполняет виртуальную симуляцию. Фигура временно перемещается на потенциальную клетку, вызывается метод `is_in_check()`. Если король оказывается под боем, ход удаляется из списка возможных, после чего доска откатывается в исходное состояние. Аналогичная проверка реализована для полей, через которые король проходит при рокировке.

### Система покадровых анимаций
Анимации вынесены в `view` для сохранения чистоты модели:
1. **Скольжение (Sliding):** При совершении хода вычисляются стартовые и конечные пиксельные координаты. С помощью линейной интерполяции (Lerp) фигура плавно перемещается между ними за заданное количество кадров (например, 6 кадров для высокой динамики).
2. **Анимация взятия (Fade-out):** Срубленная фигура остается на экране, но её альфа-канал (прозрачность) линейно уменьшается до нуля за 15-20 кадров, создавая эффект плавного исчезновения под атакующей фигурой.

### Адаптивный рендеринг интерфейса
Игра поддерживает динамическое изменение размера окна. При перехвате события `pygame.VIDEORESIZE` система пересчитывает минимальное измерение (`min(width, height)`), вычисляет новые размеры отступов, клеток и шрифтов, перестраивает UI-кнопки и масштабирует изображения фигур с использованием `pygame.transform.smoothscale`, сохраняя идеальные пропорции доски всегда в центре экрана.

## 5. Инструкция по запуску
### Требования
- Python 3.8 или выше.
- Установленная библиотека Pygame.

Установка зависимостей:
```bash
pip install pygame
```

### Запуск локальной игры
Находясь в корневой директории проекта, выполните команду:
```bash
python source/main.py
```

### Запуск онлайн-игры
1. Запустите локальный сервер (в отдельном окне терминала):
```bash
python server.py
```
2. Откройте два экземпляра клиента (на одном или разных компьютерах в локальной сети):
```bash
python source/main.py
```
3. В обоих клиентах выберите пункт **"Онлайн игра"** в главном меню.

### Управление в игре
- **Левая кнопка мыши** - выбор фигуры и совершение хода.
- **Колесико мыши** - вертикальная прокрутка текста в разделе "Справка".
- Управление размером окна производится стандартными средствами операционной системы (растягивание за углы/грани).
