# Лабораторная работа №2: Приложение для учёта футбольных команд

**Документация GUI-приложения для управления базой данных футболистов (Вариант 9)**

## 1. Общее описание
Данная программная система представляет собой десктопное приложение для ведения базы данных футболистов. Приложение позволяет хранить информацию о ФИО, дате рождения, команде, родном городе, составе и позиции игрока.

**Ключевые возможности:**
* Постраничный вывод данных (пагинация) из локальной БД SQLite.
* Многокритериальный поиск и удаление записей (согласно Варианту 9).
* Иерархическое отображение данных в виде дерева.
* Работа с внешними файлами: сохранение (DOM) и загрузка (SAX) данных в формате XML.
* Автоматическая генерация 50 реалистичных тестовых записей.

Проект реализован на языке Python с использованием библиотеки PyQt6. Архитектура приложения следует паттерну Model-View-Controller (MVC) с пассивной моделью.

## 3. Описание модулей

### 3.1. main.py
Запускает приложение. Инициализирует компоненты MVC и передает управление объекту `Controller`.

### 3.2. Пакет model
* **Player.py**: Содержит класс-сущность `Player`. Описывает структуру данных одного футболиста.
* **PlayerDatabase.py**: Реализует хранение данных в SQLite. Отвечает за SQL-запросы (`SELECT` с `LIMIT/OFFSET` для пагинации, `INSERT`, `DELETE`).
* **XmlUtils.py**: Содержит логику сериализации данных. Использует DOM для записи в файл и SAX (через `PlayerHandler.py`) для эффективного чтения больших XML-файлов.

### 3.3. Пакет view
* **MainWindow.py**: Главный экран. Содержит таблицу (`QTableWidget`), кнопки навигации по страницам, меню и панель инструментов с объектами `QAction`.
* **AddPlayerDialog.py**: Форма ввода данных с использованием `QDateEdit` (календарь) и `QComboBox`.
* **SearchDialog.py** & **DeleteDialog.py**: Динамические интерфейсы, использующие `QStackedWidget`. Позволяют переключать поля ввода в зависимости от выбранного критерия Варианта 9.

### 3.4. Пакет controller
* **Controller.py**: «Мозг» приложения. Связывает сигналы интерфейса (нажатия кнопок) с методами модели, управляет состоянием пагинации (`current_page`), обрабатывает логику генерации данных через библиотеку `Faker` и выводит системные уведомления через `QMessageBox`.

## 4. Ключевые алгоритмы и реализация

* **Паттерн MVC (Passive Model):** Приложение разделено на три независимых слоя. Модель является «пассивной» — она не знает об интерфейсе и только возвращает данные по запросу контроллера. Контроллер сам инициирует обновление View после получения данных.
* **Постраничная навигация (Pagination):** Для предотвращения перегрузки памяти при работе с большими списками реализована пагинация на уровне базы данных. SQL-запросы используют конструкцию `LIMIT ? OFFSET ?`, запрашивая только те 10-100 записей, которые необходимы для текущей страницы.
* **Динамический интерфейс (UI Transformation):** В диалогах поиска и удаления реализована сложная логика переключения виджетов через `QStackedWidget`. Это позволяет в одном окне объединить ввод текстовых данных, выбор из списков и работу с датами, сохраняя компактность интерфейса.
* **Генерация данных:** Метод `generate_test_data` использует библиотеку `Faker` (локализация `ru_RU`) для быстрого наполнения базы реалистичными именами, городами и датами рождения футболистов для тестирования функций поиска и сортировки.

* PyQt6
* Faker (для генерации данных)
